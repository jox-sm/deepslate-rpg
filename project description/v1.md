## Authentication Pattern (Clerk + Convex)

### Provider Chain (layout.tsx)
```
ClerkProvider
  ‚Üí ConvexClientProvider (ConvexProviderWithClerk + useAuth from @clerk/nextjs)
    ‚Üí AuthGate (Convex Authenticated / Unauthenticated)
      ‚Üí Page content
```

### Auth Gate (auth-gate.tsx)
- `AuthLoading` ‚Üí full-screen black spinner
- `Unauthenticated` ‚Üí renders ONLY the auth screen (no page content at all)
- `Authenticated` ‚Üí renders children (sidebar, games, etc.)

### Login / Signup Components
- Use `useClerk()` hook directly for button clicks:
  - `clerk.openSignIn({})` ‚Äî opens Clerk sign-in modal
  - `clerk.openSignUp({})` ‚Äî opens Clerk sign-up modal
  - Wrapped in Convex `<Unauthenticated>` so they auto-hide when authenticated

### Setup Requirements
1. Activate Convex integration at `https://dashboard.clerk.com/apps/setup/convex`
2. Set `domain` in `convex/auth.config.ts` to the Clerk Frontend API URL
3. Push config: `npx convex dev --once`
4. Set `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` and `CLERK_SECRET_KEY` in `.env.local`
5. User must sign out completely and sign back in for fresh Convex-compatible token

## Convex Architecture

### Schema (convex/schema.ts)
- **games**: name, description, image?, tags[], likesCount ‚Äî indexes: by_name, by_likes_count
- **characters**: gameId (FK‚Üígames), name, description, image? ‚Äî index: by_game_id
- **maps**: gameId (FK‚Üígames), name, image?, sizeOfPlace?, placesAtMap? ‚Äî index: by_game_id
- **items**: gameId (FK‚Üígames), name, image? ‚Äî index: by_game_id

### Function Patterns
- **Always use validators** on args ‚Äî no exceptions
- **Public functions**: `query` / `mutation` from `./_generated/server`
- **Private functions**: `internalQuery` / `internalMutation` for non-API functions
- **Pagination**: use `paginationOptsValidator` + `.paginate()` for list queries
- **Bounded results**: use `.take(n)` instead of `.collect()` ‚Äî never unbounded
- **Index queries**: use `.withIndex()` never `.filter()`
- **Updates**: use `.patch()` for partial updates, `.replace()` for full replaces
- **No `.delete()` on queries**: batch read + iterate `ctx.db.delete(row._id)`
- **File-based routing**: `convex/games.ts` ‚Üí `api.games.*`, `convex/characters.ts` ‚Üí `api.characters.*`

## Data Flow

### Game Creation Pipeline
```
User ‚Üí CreateForm (name, desc, tags, image)
  ‚Üí Wizard: Characters step ‚Üí Maps step ‚Üí Items step ‚Üí Submit
  1. POST /api/convertUrl ‚Üí main image ‚Üí Supabase Storage ‚Üí URL
  2. fileToBase64() ‚Üí all component images ‚Üí base64
  3. POST /api/push {type:game} ‚Üí Redis queue "InsertGames"
     ‚Üí pull.ts processes ‚Üí Neon PostgreSQL INSERT
  4. POST /api/convertUrl/ConvertGameImages ‚Üí base64 ‚Üí Supabase URLs
  5. POST /api/push/pushGames ‚Üí Redis queue "InsertGamesmongodb"
     ‚Üí processGamesQueue() ‚Üí MongoDB Game.insertMany()
```

### Game Reading Pipeline
```
GET /api/games?page=&limit=
  ‚Üí ensureCachePrimed() ‚Üí Redis cached IDs from sorted set
  ‚Üí Offset in cache: return cached (source: "redis")
  ‚Üí Fallback: PostgreSQL paginated (source: "postgresql")
```

## Key Gotchas

1. **Next.js 16 is NOT standard** ‚Äî breaking changes in file conventions, APIs. Always check `node_modules/next/dist/docs/` first.
2. **Convex `auth.config.ts` `domain` key** ‚Äî must be the exact Clerk Frontend API URL, not a placeholder
3. **Clerk + Convex requires fresh sign-in** after configuring the Convex integration ‚Äî old tokens don't have the right audience claim
4. **`ConvexProviderWithClerk` expects `useAuth` from `@clerk/nextjs`** ‚Äî not the raw hook, the export
5. **Convex `<Authenticated>` / `<Unauthenticated>`** ‚Äî these are Convex's dynamic auth components from `convex/react`, not Clerk's
6. **Convex functions always need validators** ‚Äî missing validators = runtime error
7. **`npx convex dev --once`** ‚Äî use this to push code and validate schema, NOT `npx convex deploy` (that's for production)
8. **`"clerk"` in package.json is the CLI tool** ‚Äî the actual auth packages are `@clerk/nextjs`, `@clerk/react`, etc.
9. **PostHog reverse-proxied** via Next.js rewrites (`/ingest/*` ‚Üí `us.i.posthog.com/*`)
10. **No global state management** ‚Äî 100% local state via hooks, props drilled through components
